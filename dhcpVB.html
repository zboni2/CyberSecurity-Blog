Setting Up a Local Hacking Lab with VirtualBox

Written by Altin Hajdini
November 2, 2025 5 min read

Creating a secure, isolated hacking lab is essential for practicing penetration testing, working on TryHackMe challenges, or learning cybersecurity in a safe environment. This guide will show you how to set up your own local network within VirtualBox, completely isolated from your host system.

Prerequisites

  • Windows operating system
  • Oracle VirtualBox installed
  • At least two virtual machines (e.g., Kali Linux and a target VM)

Setting Up the Internal Network

Step 1: Open Command Prompt as Administrator

Press Windows + X and select "Windows Terminal (Admin)" or "Command Prompt (Admin)"

Step 2: Navigate to VirtualBox Directory

Run the following command to change to the VirtualBox installation directory:

cd "C:\Program Files\Oracle\VirtualBox"

Step 3: Create a DHCP Server

Execute the following command to set up a DHCP server for your internal network:

vboxmanage dhcpserver add --network="godofwar" --server-ip="10.21.0.1" --lower-ip="10.21.0.100" --upper-ip="10.21.0.110" --netmask="255.255.255.0" --enable
Command Breakdown:
  • --network="godofwar": Name of your internal network
  • --server-ip="10.21.0.1": IP address that will act as the router
  • --lower-ip="10.21.0.100": Starting IP in the DHCP range
  • --upper-ip="10.21.0.110": Ending IP in the DHCP range
  • --netmask="255.255.255.0": Subnet mask for the network

Configuring Virtual Machines

For each VM (Kali Linux and Target VM):

  1. Right-click on the VM and select Settings
  2. Go to Network settings
  3. Under Adapter 1, change "Attached to:" to Internal Network
  4. In the dropdown, select the network name you created (e.g., "HackLabNet")
  5. Click OK to save the settings

Verifying the Setup

1. Start both VMs

Boot up your Kali Linux and target VMs. They should automatically receive IP addresses from the DHCP server.

2. Check Network Connectivity

In Kali Linux, open a terminal and run:

ip a

You should see an IP address in the range you specified (e.g., 10.21.0.100-110).

3. Test Connection Between VMs

From Kali Linux, try to ping your target VM:

ping 10.21.0.101

Benefits of This Setup

  • Isolated Environment: Your hacking lab is completely separated from your host system and the internet
  • Safe for Testing: No risk of accidentally scanning or attacking real networks
  • Consistent Network: IP addresses remain the same between reboots
  • No Internet Required: Work on your lab even without an internet connection

Troubleshooting

No IP Address Assigned

If your VMs don't get an IP address:

  1. Verify the DHCP server is running: vboxmanage list dhcpservers
  2. Check the network adapter settings in VirtualBox
  3. Try restarting the VMs

Cannot Ping Between VMs

  • Ensure both VMs are on the same internal network
  • Check the firewall settings on both VMs
  • Verify the network adapter is enabled in the VM settings

Conclusion

Nice now you've successfully created an isolated hacking lab environment using VirtualBox's internal networking. This setup is perfect for practicing penetration testing, working on CTF challenges, or learning about network security in a safe, controlled environment. Remember to always use these skills ethically and legally.
